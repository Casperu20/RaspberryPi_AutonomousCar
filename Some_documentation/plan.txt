For a heatmap, every measurement needs a position:

The simplest solution is to define the room as a matrix.

The robot starts at a known cell, for example:

robot_x = 10

NORTH -> y - 1

    NORTH: (0, -1),
    EAST:  (1, 0),
    SOUTH: (0, 1),
    WEST:  (-1, 0),

Then after a successful forward movement:

dx, dy = DIRECTIONS[robot_heading]
robot_x += dx
robot_y += dy

Level 2: Occupancy Grid

Each cell in the matrix can be:

? ? ? ? ?

? ? . ? ?
? ? ? ? ?

The robot uses its ultrasonic sensors to update nearby cells.

For example, if the front sensor sees an obstacle closer than one cell:

If front is clear:

then mark the front cell as:

FREE

Same idea for left and right sensors, but you translate left/right relative to the robot’s heading.

Simple idea:

This is a good strategy for your robot.

Option C: Frontier Exploration

This is the more advanced and better robotics strategy.

A “frontier” is a free cell next to an unknown cell.

Example:

Known free area next to unknown area = frontier

The robot repeatedly chooses the nearest frontier and goes there.

Pseudo-logic:

while True:

closer to real autonomous mapping

Cons:

I would not start here immediately, but this is where the project should go.

Important Note About DHT

A DHT sensor gives you:

temperature
humidity

It does not measure air quality by itself.

For real air quality, you probably want one of these later:

BME680       -> temperature, humidity, pressure, gas resistance
SGP30        -> VOC / eCO2 estimate
CCS811       -> VOC / eCO2 estimate
PMS5003      -> particulate matter PM1.0 / PM2.5 / PM10
MQ135        -> gas sensor, cheaper but less precise

A DHT sensor is still useful because humidity and temperature affect air-quality readings, but for an “air quality heatmap,” DHT alone is not enough.

Recommended Camera Strategy: ArUco Markers

For your project, I would use ArUco markers.

ArUco markers are printed square markers that OpenCV can detect. OpenCV’s ArUco module is specifically used for marker detection and pose estimation, which is useful for robot navigation and localization.

You would print markers like this:

Marker 1 -> wall near grid cell (0, 0)
Marker 2 -> wall near grid cell (5, 0)
Marker 3 -> wall near grid cell (5, 5)
Marker 4 -> wall near grid cell (0, 5)

Then in code:

known_markers = {
    1: {"x": 0, "y": 0, "heading": "NORTH"},
    2: {"x": 5, "y": 0, "heading": "EAST"},
    3: {"x": 5, "y": 5, "heading": "SOUTH"},
    4: {"x": 0, "y": 5, "heading": "WEST"},
}

When the camera sees marker 2, the robot can say:

I am near the right side of the room.
My position estimate should be corrected.

This gives you a much more reliable map than only motor timing.
